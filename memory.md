# Workflow Memory

- When the user asks for repo work from `main` or another base branch, create a separate working branch before editing.
- If the user asks for Korean git communication, use Korean commit messages, PR titles, PR bodies, and review comments.
- Prefer multiple small commits instead of one large commit when the user explicitly asks for it.
- If the user asks to reflect the same change into `develop`, prepare the corresponding `develop` branch and PR as part of the same task when feasible.
